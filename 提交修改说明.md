# 提交代码说明(简明版)

> **KK的奇妙冒险** · 推送到 GitHub 仓库的最快路径
> 远程:`https://github.com/asce-12138/KK-s_Wonder_Adventure.git` · 分支:`main`

---

## 🚀 标准流程(5 步)

```powershell
# 0. 进入项目目录
cd KK-s_Wonder_Adventure

# 1. 查看当前状态(红色 = 已修改,绿色 = 已暂存)
git status

# 2. 把改动加入暂存区
git add .                                    # 提交所有
git add src/main.py                          # 或只提交某个文件

# 3. 提交到本地
git commit -m "feat: 简述本次改动"

# 4. 推送到 GitHub
git pull origin main                         # 先拉取,避免冲突
git push origin main                         # 再推送
```

> 🔑 **第一次推送会要求输入 GitHub 用户名 + Personal Access Token**(不是账号密码)。Token 生成方法见下方「Token 配置」。

---

## ⚙️ Token 配置(只做一次)

1. 打开 https://github.com/settings/tokens
2. **Generate new token** → **Fine-grained token**
3. 设置:
   - Token name:随意
   - Repository access:勾选 `KK-s_Wonder_Adventure`
   - Permissions → Repository permissions → **Contents**:**Read and write**
4. 生成后**立即复制** token
5. 推送时,密码处粘贴这个 token 即可

## 🔧 必做配置(只做一次)

```powershell
# 1. 提交人信息
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"

# 2. 远程地址(确保是 HTTPS,不是 SSH)
git remote set-url origin https://github.com/asce-12138/KK-s_Wonder_Adventure.git

# 3. 确认设置正确
git remote -v
```

应输出:
```
origin  https://github.com/asce-12138/KK-s_Wonder_Adventure.git (fetch)
origin  https://github.com/asce-12138/KK-s_Wonder_Adventure.git (push)
```

## ❗ 常见报错速解

| 报错 | 原因 | 解决 |
|------|------|------|
| `ssh: connect to host github.com port 22: Connection refused` | 当前用了 SSH 协议,22 端口被封 | `git remote set-url origin https://github.com/asce-12138/KK-s_Wonder_Adventure.git` |
| `! [rejected] main -> main (fetch first)` | 远程有更新,本地落后 | 先 `git pull origin main`,再 `git push` |
| `Authentication failed` | token 输错 / 过期 | 重新生成 token 再粘贴 |
| 冲突标记 `<<<<<<<` / `=======` / `>>>>>>>` | 多人在同一文件改代码 | 手动编辑文件保留正确内容,删掉标记,再 `git add .` + `git commit` + `git push` |
| 误提交了 `.venv` | `git add .` 把虚拟环境也加进去了 | `git rm --cached -r .venv` 后再 commit(`.gitignore` 已自带忽略) |
